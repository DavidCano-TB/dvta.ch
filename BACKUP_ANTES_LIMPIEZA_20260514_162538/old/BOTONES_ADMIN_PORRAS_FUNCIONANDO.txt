═══════════════════════════════════════════════════════════════════════════
  BOTONES DISPONIBLES Y SU FUNCIONALIDAD
═══════════════════════════════════════════════════════════════════════════

1. 🔒 CERRAR APUESTAS
   ───────────────────
   Endpoint: POST /api/porras/cerrar/{porra_id}
   Función: Cierra la porra para que no se puedan hacer más apuestas
   Permisos: DVD o creador de la porra
   Visible cuando: Estado = 'abierta'
   
   Comportamiento:
   - Cambia el estado de 'abierta' a 'cerrada'
   - Registra la fecha de cierre (closed_at)
   - No se pueden hacer más apuestas
   - El bote queda congelado

2. ✓ RESOLVER Y PAGAR
   ──────────────────
   Endpoint: POST /api/porras/resolver
   Función: Resuelve la porra seleccionando el resultado ganador y paga premios
   Permisos: Solo DVD
   Visible cuando: Estado = 'cerrada'
   
   Comportamiento:
   - Solicita seleccionar la opción ganadora
   - Calcula las ganancias proporcionalmente
   - Paga automáticamente a todos los ganadores
   - Actualiza estadísticas de usuarios
   - Registra transacciones de premios
   - Cambia el estado a 'finalizada'
   - Muestra panel de resultados con ganadores

3. ✕ CANCELAR Y DEVOLVER
   ──────────────────────
   Endpoint: POST /api/porras/cancelar/{porra_id}
   Función: Cancela la porra y devuelve el dinero a todos los apostadores
   Permisos: DVD o creador de la porra
   Visible cuando: Estado != 'finalizada'
   
   Comportamiento:
   - Devuelve el 100% del dinero apostado a cada usuario
   - Actualiza los saldos automáticamente
   - Marca todas las apuestas como pagadas (ganancia = 0)
   - Cambia el estado a 'cancelada'
   - Muestra número de apuestas devueltas

4. 🔄 RELANZAR PORRA
   ─────────────────
   Endpoint: POST /api/porras/relanzar/{porra_id}
   Función: Reabre una porra cerrada o finalizada con nuevas fechas
   Permisos: Solo DVD
   Visible cuando: Estado = 'cerrada' o 'finalizada'
   
   Comportamiento:
   - Solicita nueva fecha límite
   - Solicita nueva fecha del evento
   - Valida que las fechas sean futuras y coherentes
   - Cambia el estado a 'abierta'
   - Limpia resultado anterior
   - Permite nuevas apuestas

5. 👁️ ENMASCARAR/OCULTAR
   ─────────────────────
   Endpoint: POST /api/porras/enmascarar/{porra_id}
   Función: Oculta o muestra la porra en la lista pública
   Permisos: Solo DVD
   Visible: Siempre (para DVD)
   
   Comportamiento:
   - Pregunta si ocultar o mostrar
   - Actualiza el campo 'enmascarada' en la base de datos
   - Las porras enmascaradas no aparecen en /apuestas
   - Se pueden acceder directamente por URL
   - Útil para porras privadas o de prueba

6. 🗑 BORRAR PORRA
   ────────────────
   Endpoint: DELETE /api/porras/{porra_id}
   Función: Elimina permanentemente la porra de la base de datos
   Permisos: DVD (siempre) o creador (solo si no hay apuestas)
   Visible cuando: 
     - DVD: Siempre
     - Creador: Solo si total_apostadores = 0
   
   Comportamiento:
   - Solicita doble confirmación
   - Verifica que no haya apuestas (si las hay, debe cancelar primero)
   - Elimina la porra permanentemente
   - Redirige a /apuestas
   - ACCIÓN IRREVERSIBLE

  FLUJO DE TRABAJO TÍPICO
═══════════════════════════════════════════════════════════════════════════

ESCENARIO 1: Porra Normal
1. Crear porra (cualquier usuario)
2. Usuarios apuestan mientras está 'abierta'
3. DVD cierra apuestas → Estado: 'cerrada'
4. Ocurre el evento
5. DVD resuelve y paga → Estado: 'finalizada'
6. Ganadores reciben premios automáticamente

ESCENARIO 2: Porra Cancelada
1. Crear porra
2. Usuarios apuestan
3. Evento se cancela o hay problema
4. DVD o creador cancela la porra
5. Todos reciben su dinero de vuelta
6. Estado: 'cancelada'

ESCENARIO 3: Porra Relanzada
1. Porra cerrada o finalizada
2. DVD decide relanzarla
3. Introduce nuevas fechas
4. Estado vuelve a 'abierta'
5. Se pueden hacer nuevas apuestas

ESCENARIO 4: Porra Privada
1. DVD crea porra
2. DVD enmascara la porra
3. Porra no aparece en lista pública
4. Solo accesible por URL directa
5. Útil para porras privadas o de prueba
