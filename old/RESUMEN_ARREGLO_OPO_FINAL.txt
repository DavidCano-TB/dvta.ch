🐛 PROBLEMA ORIGINAL
═══════════════════════════════════════════════════════════

  "En la pantalla de management de OPO, los test no arranca
  tras apretar uno de los botones"

  Mensaje de error:
  "Conectando al servidor... Por favor espera un momento e
  intenta de nuevo"


🔍 CAUSA RAÍZ IDENTIFICADA
═══════════════════════════════════════════════════════════

  1. WebSocket de OPO requiere autenticación
  2. El código no enviaba el token de sesión
  3. Sin token, el servidor rechaza la conexión
  4. El usuario no estaba logueado


✅ SOLUCIÓN IMPLEMENTADA
═══════════════════════════════════════════════════════════

  CAMBIO 1: Agregada función getCookie()
  ───────────────────────────────────────
    • Lee el token de las cookies del navegador
    • Función JavaScript simple y efectiva

  CAMBIO 2: WebSocket envía token en URL
  ───────────────────────────────────────
    • URL: ws://localhost:8000/ws/opo?token=XXX
    • El servidor valida el token
    • Si es válido, conecta; si no, rechaza

  CAMBIO 3: Verificación de autenticación
  ────────────────────────────────────────
    • Verifica /api/opo/status antes de conectar
    • Si no está autenticado, muestra pantalla de login
    • Si no tiene acceso, redirige a inicio

  CAMBIO 4: Manejo de códigos de error
  ─────────────────────────────────────
    • 4001: Auth required → Redirige a /
    • 4003: Access denied → Muestra mensaje
    • Otros: Intenta reconectar

  CAMBIO 5: Logs detallados
  ─────────────────────────
    • Log de usuario autenticado
    • Log de URL de conexión (token oculto)
    • Log de códigos de cierre
    • Mejor debugging
