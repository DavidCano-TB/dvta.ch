🔍 PROBLEMA DIAGNOSTICADO:
--------------------------
El sistema NO detectaba ningún personaje porque la API key de Gemini estaba
configurada para usar el modelo "gemini-pro-latest" (gemini-3.1-pro), pero
la CUOTA GRATUITA de este modelo estaba AGOTADA (limit: 0).

Error específico:
  "Quota exceeded for metric: generate_content_free_tier_input_token_count,
   limit: 0, model: gemini-3.1-pro"


✅ SOLUCIÓN APLICADA:
---------------------
1. Cambio de modelo de IA en ai_helper.py:
   ANTES: gemini-pro-latest (cuota agotada)
   AHORA: gemini-2.5-flash (cuota disponible)

2. Cambio de API:
   ANTES: v1beta
   AHORA: v1

3. Mejoras en manejo de errores:
   - Manejo de corrected_name: null
   - Manejo de main_known_for: null
   - Mejor detección de JSON incompleto

🔧 TROUBLESHOOTING:
-------------------
Si la IA no responde:
  1. Verificar que existe: config/.gemini_key
  2. Verificar cuota en: https://ai.dev/rate-limit
  3. Ejecutar: python TEST_GEMINI_QUIEN_SOY.py

Si aparece error 429 (Rate Limit):
  - Esperar 1 minuto (límite: 5 requests/minuto en tier gratuito)

Si aparece error 404 (Model Not Found):
  - Ejecutar: python LIST_GEMINI_MODELS.py
  - Cambiar modelo en ai_helper.py
